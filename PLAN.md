# Chisel — Lightweight PDF Editor for GNOME

**App ID**: `com.uncapyeti.chisel`
**Binary**: `chisel`
**Desktop file**: `com.uncapyeti.chisel.desktop`

A fast, native GNOME PDF editor built for telecom tower mapping reports.
Focused on text editing (including stacked fractions), annotations,
image insertion, and page management.

## Tech Stack

| Component | Choice | Why |
|---|---|---|
| UI Toolkit | GTK 4 + libadwaita | Native GNOME look |
| Language | Python 3 + PyGObject | Fast development |
| PDF Rendering | PyMuPDF (fitz) | Fastest rendering, text extraction, redaction |
| PDF Annotations | Poppler (via GIR) | Standard PDF annotation support |
| Drawing/Canvas | Cairo (via GTK DrawingArea) | Vector overlay drawing |

## Target PDFs

Telecom tower mapping reports generated by AutoCAD 2025:
- 16-17 pages per report
- Sections: Index, Abbreviations, Site Info, Tower Info, Tower Elevation,
  Base Details, Tower Geometry, Steel Data, Guy Anchor/Details,
  Torque Arm, Climbing Ladders, Coax Plan, Antenna Details, Photos, Notes
- Fractions rendered as stacked spans (numerator, bar, denominator)
- Dimension labels attached to line drawings
- Tables for steel data, coax routing, antenna specs
- Embedded JPEG tower photos

## The Fraction Problem

AutoCAD exports fractions as stacked text elements:
- Numerator text span (e.g., "1")
- Denominator text span (e.g., "4")
- Horizontal line drawn between them

Text extraction sees these as separate tokens:
```
Extracted:  1 4"      → Should be:  1/4"
Extracted:  3 16"     → Should be:  3/16"
Extracted:  7 8"Ø     → Should be:  7/8"Ø
Extracted:  212"      → Should be:  2-1/2"
Extracted:  61 4"     → Should be:  6-1/4"
```

Strategy: detect clustered spans that form a fraction pattern, group
them as a single editable unit, show a fraction editor widget.
